## Задача 1. Техническое задание для программы «Перевод времени»

**1. Назначение программы:**  
Программа переводит время из часов в минуты.

**2. Входные данные:**  
* Тип данных: число (целое или дробное).  
* Диапазон значений: от 0 до 1000 часов.  
* Формат ввода: с клавиатуры или через интерфейс программы.  

**3. Выходные данные:**  
* Тип данных: целое число.  
* Единица измерения: минуты.  
* Формат вывода: число без дополнительных символов (например, 120).  

**4. Алгоритм работы:**  
1. Получить значение времени в часах (h).  
2. Умножить значение на 60: m = h * 60.  
3. Вывести результат (m).  

**5. Обработка ошибок:**  
* Если введено отрицательное число — вывести сообщение: «Ошибка: время не может быть отрицательным».  
* Если введено не число — вывести сообщение: «Ошибка: введите корректное число».  

**6. Требования к интерфейсу:**  
* Простое текстовое поле для ввода часов.  
* Кнопка «Перевести».  
* Область для отображения результата.

## Задача 2. Техническое задание для программы «Сравнение чисел»

**1. Назначение программы:**  
Программа сравнивает два числа и определяет, какое из них больше, меньше или равны ли они.

**2. Входные данные:**  
* Два числа (целые или дробные).  
* Диапазон: от -10 000 до 10 000.  
* Формат ввода: два отдельных поля в интерфейсе.  

**3. Выходные данные:**  
* Текст с результатом сравнения:  
    * «Первое число больше второго»,  
    * «Второе число больше первого»,  
    * «Числа равны».  

**4. Алгоритм работы:**  
1. Получить первое число (a).  
2. Получить второе число (b).  
3. Сравнить a и b:  
    * Если a > b — вывести «Первое число больше второго».  
    * Если a < b — вывести «Второе число больше первого».  
    * Если a = b — вывести «Числа равны».  

**5. Обработка ошибок:**  
* При вводе нечисловых данных — сообщение: «Ошибка: введите числа».  

**6. Требования к интерфейсу:**  
* Два поля для ввода чисел.  
* Кнопка «Сравнить».  
* Область для вывода результата.

## Задача 3. Техническое задание для программы «Допуск к экзамену»

**1. Назначение программы:**  
Проверка допуска студента к экзамену на основе посещаемости и выполнения заданий.

**2. Входные данные:**  
* Посещаемость (в %): от 0 до 100.  
* Процент выполненных заданий: от 0 до 100.  

**3. Условия допуска:**  
* Посещаемость >= 75 %.  
* Выполнено >= 80 % заданий.  
Если оба условия выполнены — допуск разрешён.  

**4. Выходные данные:**  
* «Допущен к экзамену» или «Не допущен к экзамену».  

**5. Алгоритм работы:**  
1. Ввести посещаемость (p).  
2. Ввести процент выполненных заданий (z).  
3. Проверить условия:  
    * Если p >= 75 и z >= 80 — вывести «Допущен к экзамену».  
    * Иначе — вывести «Не допущен к экзамену».  

**6. Обработка ошибок:**  
* Значения вне диапазона [0; 100] — сообщение: «Ошибка: введите значения от 0 до 100».  
* Нечисловые данные — сообщение: «Ошибка: введите числа».  

**7. Требования к интерфейсу:**  
* Поле для ввода посещаемости.  
* Поле для ввода процента выполненных заданий.  
* Кнопка «Проверить допуск».  
* Область вывода результата.

## Задача 4. Техническое задание для программы «Скидка в магазине»

**1. Назначение программы:**  
Расчёт итоговой стоимости покупки с учётом скидок.

**2. Входные данные:**  
* Сумма покупки (S) в рублях: от 0 до 1 000 000.  
* Статус клиента (опционально): «постоянный» / «новый».  

**3. Правила скидок:**  
* S < 1000 руб. — скидка 0 %.  
* 1000 <= S < 5000 руб. — скидка 5 %.  
* S >= 5000 руб. — скидка 10 %.  
* Для постоянных клиентов — дополнительная скидка 5 % от итоговой суммы.  

**4. Выходные данные:**  
* Итоговая стоимость с учётом скидок (в рублях, округлённая до копеек).  

**5. Алгоритм работы:**  
1. Ввести сумму покупки (S).  
2. Выбрать статус клиента.  
3. Рассчитать скидку:  
    * По сумме покупки.  
    * Добавить скидку для постоянных клиентов (если применимо).  
4. Вычислить итоговую стоимость: S_итог = S * (1 - скидка).  
5. Вывести S_итог.  

**6. Обработка ошибок:**  
* Отрицательные суммы — сообщение: «Ошибка: сумма не может быть отрицательной».  
* Некорректный статус — сообщение: «Выберите статус клиента».  

**7. Требования к интерфейсу:**  
* Поле ввода суммы.  
* Выпадающий список для статуса клиента.  
* Кнопка «Рассчитать».  
* Отображение скидки и итоговой суммы.

## Задача 5. Техническое задание для программы «Квадратное уравнение»

**1. Назначение программы:**  
Решение квадратного уравнения вида ax^2 + bx + c = 0.

**2. Входные данные:**  
* Коэффициенты a, b, c (целые/дробные числа).  
* Ограничения: a != 0, диапазон [-1000; 1000].  

**3. Выходные данные:**  
* Корни уравнения (если существуют).  
* Сообщение:  
    * «Уравнение не имеет действительных корней» (при D < 0),  
    * «Один корень: x = ...» (при D = 0),  
    * «Два корня: x1 = ..., x2 = ...» (при D > 0).  

**4. Алгоритм работы:**  
1. Ввести a, b, c.  
2. Вычислить дискриминант: D = b^2 - 4*a*c.  
3. Проанализировать D:  
    * D < 0 → вывод сообщения об отсутствии корней.  
    * D = 0 → x = (-b) / (2*a).  
    * D > 0 → x1 = (-b + sqrt(D)) / (2*a), x2 = (-b - sqrt(D)) / (2*a).  
4. Вывести результат.  

**5. Обработка ошибок:**  
* a = 0 → сообщение: «Ошибка: коэффициент a не может быть равен нулю».  
* Нечисловые данные → сообщение: «Введите корректные числа».  

**6. Требования к интерфейсу:**  
* Три поля для коэффициентов.  
* Кнопка «Решить».  
* Область вывода корней или сообщения.  
* Возможность сброса полей.
